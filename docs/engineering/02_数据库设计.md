# 数据库设计

## 状态

本文是 PostgreSQL + pgvector 第一版主链路的表设计。当前已有 SQLAlchemy models 和冻结 Alembic 初始迁移；离线 SQL 已验证，真实 PostgreSQL 在线迁移未验证。

## 数据库原则

- 主数据库使用 PostgreSQL。
- 第一版启用 pgvector，向量记忆直接在 PostgreSQL 内落库和检索。
- LangGraph checkpointer 使用 PostgreSQL。
- SQLite 只允许个人临时实验，不进入验收主路径。
- 所有外部回调、副作用和 resume 必须有数据库幂等键。
- artifact 和 memory 的全文不放 LangGraph state；state 只保存摘要和引用。
- 所有时间字段使用 `TIMESTAMPTZ`。
- 主键默认 `UUID`，外部系统 ID 保留原始字符串。
- 飞书事件和卡片去重必须使用 `event_id`、`message_id`、`action_id` 和业务 `idempotency_key`，防止重复启动 graph、重复 resume 或重复 Bitable 写入。
- Discord interaction 表只作为 optional adapter 规划保留；后续启用时仍必须遵守相同幂等规则。

## 扩展

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | UUID PK | profile id |

| `user_id` | TEXT NOT NULL | channel user id；Feishu First 中可对应 open_id/union_id/user_id |
| `provider` | TEXT NOT NULL | openai / deepseek |

| `base_url` | TEXT NULL | provider base URL，默认按 provider 决定 |
| `usage` | TEXT NOT NULL | chat / embedding |
| `status` | TEXT NOT NULL | active / revoked |
| `is_default` | BOOLEAN NOT NULL | 当前用户该 usage 的默认 profile |

- `UNIQUE(guild_id, user_id, display_name)`

安全规则：
- 查询列表只能返回 `ModelProfileSummary`，不得返回 `encrypted_api_key`。
- 运行时必须同时校验 `profile_id + guild_id + user_id`，防止用户 A 使用用户 B 的 profile；Feishu First 后续可增加别名字段或包装层，但数据库兼容字段暂不破坏。
- OpenAI / DeepSeek chat profile 可用于 LLMGateway；embedding profile 第一版只允许 OpenAI。
- 无 embedding profile 时，MemoryGateway 向量检索/写入降级为 skipped，不得借用其他用户 key。

- `idx_memory_owner_status(owner_agent, status)`
- `idx_memory_source_run_id(source_run_id)`
- `idx_memory_expires_at(expires_at)`
- `idx_memory_renewal_required_at(renewal_required_at)`

状态规则：
- `RunMemory` 可自动写入并进入 `active`，默认 `retention_policy='30d'`，但仍只以摘要/ref 注入 ContextPack。
- `ProjectMemory`、`AgentMemory`、`CaseMemory`、`UserCorrectionMemory` 初始为 `draft` 或 `pending_review`，默认 `retention_policy='90d'`。
- `ProceduralMemory` / SOP 类记忆默认 `retention_policy='permanent'`，但必须版本化和可撤销。
- 只有 `memory_update_proposals.status='approved'` 后，长期记忆才可置为 `active`。
- `draft`、`pending_review`、`revoked`、`expired` 不得进入可检索长期记忆池。
- 检索必须先按 `thread_id / tenant_id / guild_id / user_id / project_id / requisition_id / candidate_id` 做 scope filter，再进入向量召回。
- `RunMemory` 必须带 `thread_id` scope filter；长期记忆至少必须带 `tenant_id / guild_id / user_id / project_id / requisition_id / candidate_id` 之一，否则 MemoryGateway fail closed，不执行 embedding 检索。

当前代码已实现 `tenant_id/guild_id/user_id/project_id/requisition_id/candidate_id/thread_id` scope columns、索引、SQL filter、Gateway fail closed 和 upsert FK 修复。`memory_type`、`retention_policy`、`last_hit_at`、`renewal_required_at`、ProceduralMemory enum 仍是后续 schema 扩展项，未验证。

## Checkpointer

LangGraph PostgreSQL checkpointer 建议使用独立 schema 或表前缀，例如 `langgraph_checkpoints_*`。迁移阶段必须确认官方 `langgraph-checkpoint-postgres` 当前版本要求，并在部署脚本中执行 checkpointer setup。

## 查询和事务要求

- 飞书事件 ACK / 卡片 toast 前事务必须提交 `feishu_event_logs` / `feishu_card_actions` 和 `feishu_outbox`。
- outbox worker 使用带 lease 的原子 claim，避免多个 worker 重复执行同一副作用，并能回收崩溃 worker 的 claimed 任务。
- 所有人工确认使用 `idempotency_key` 唯一约束。
- Bitable 写入必须在 HumanApproval 后执行，并使用官方 `client_token` + 内部实体到 `record_id` 的映射表做 upsert。
- 向量检索必须先按 `memory_items.status='active'`、scope、PII、owner policy 过滤，再 join embedding 查询；`pending_review`、`revoked`、`expired` 永不注入。
- 向量检索必须包含 `thread_id / tenant_id / guild_id / user_id / project_id / requisition_id / candidate_id` scope filter，并把实际 filter 写入 `memory_retrieval_audits.filters`。
- 向量检索必须过滤 `expires_at` 和 `retention_policy`，30d/90d 到期记忆进入 expired 或降权，permanent 记忆仍需可撤销。
- 公司背调当前事实、在职状态、融资新闻、裁员和组织变化不得由长期记忆直接生成，必须由 SearchGateway source_refs 支撑；长期记忆只能作为背景经验或 taxonomy 辅助。
- 大内容读取必须写入 `content_access_audits`；默认 API 只返回摘要和 ref。
