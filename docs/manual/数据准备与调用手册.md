# 数据准备与调用手册

本文说明第一版 Feishu First 需要准备什么数据，以及用户如何通过飞书调用。

## 数据类型

建议准备：

```text
岗位 JD
候选人简历或摘要
目标公司/行业列表
过往案例
用户修正和偏好
SOP / 话术模板
```

原文、大附件和长报告进入 ArtifactStore；LangGraph state 和飞书 War Room 只保存摘要、`content_ref` 和 `source_refs`。

## 字段来源规则

TaskIntakeParser 输出的关键字段必须有：

```text
field_source
confidence
```

无 source 的推断只能进入 `assumptions`，不能进入事实字段。用户在飞书确认卡 approve 后，`CanonicalTaskBrief.version` 冻结；后续修改必须 edit 生成新版本并再次确认。

## 飞书调用方式

在飞书测试群里 @机器人：

```text
@AI猎头机器人 新建岗位：上海 数据产品经理。目标：做岗位校准和候选人画像。
```

筛选候选人：

```text
@AI猎头机器人 筛选候选人。候选人摘要：... 目标岗位：...
```

触发三省六部：

```text
@AI猎头机器人 这个岗位请走三省六部完整会审。
```

系统必须返回任务确认卡，用户 approve 后才进入正式 graph。

## Bitable 同步数据

Bitable 只同步展示数据，例如：

```text
岗位作战单
人才地图摘要
候选人评估摘要
审批结果
任务状态
```

同步前必须人工确认；不得由 Agent 直接写 Bitable。

## 当前事实与长期记忆

- 公司背调、融资、裁员、组织变化等当前事实必须来自 SearchGateway `source_refs`。
- 长期记忆只提供 taxonomy、用户修正、项目规则、SOP 和历史经验。
- 简历关键词抽取以当前简历/JD artifact 为主。
