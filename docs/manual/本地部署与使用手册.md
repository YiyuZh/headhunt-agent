# 本地部署与使用手册

本文按 Feishu First 第一版主链路写：FastAPI API、worker、PostgreSQL + pgvector、Caddy HTTPS、飞书事件/卡片和 Bitable 同步。Discord 只作为 optional adapter。

## 服务器一键启动

在项目根目录：

```bash
cd /Users/w/Documents/lietou
cp .env.example .env
```

编辑 `.env`，至少填写：

```env
DOMAIN=your-domain.com
INTERNAL_ADMIN_API_KEY=change-this-admin-key
MODEL_SECRET_ENCRYPTION_KEY=change-this-model-secret

POSTGRES_DB=lietou
POSTGRES_USER=lietou
POSTGRES_PASSWORD=change-this-password
POSTGRES_HOST=postgres
POSTGRES_INTERNAL_PORT=5432

FEISHU_BASE_URL=https://open.feishu.cn
FEISHU_APP_ID=
FEISHU_APP_SECRET=
FEISHU_VERIFICATION_TOKEN=
FEISHU_ENCRYPT_KEY=
FEISHU_EVENT_CALLBACK_PATH=/feishu/events
FEISHU_CARD_ACTION_CALLBACK_PATH=/feishu/card-actions
FEISHU_DEFAULT_CHAT_ID=
FEISHU_BITABLE_APP_TOKEN=
FEISHU_BITABLE_TABLE_ID=
```

启动：

```bash
docker compose up -d --build
docker compose ps
```

健康检查：

```bash
curl https://<DOMAIN>/health
curl -H "X-Internal-Admin-Key: <INTERNAL_ADMIN_API_KEY>" https://<DOMAIN>/ready
```

如果本机没有 Docker，不能写成已通过；到服务器执行上述命令。

## 对外路径

公开路径：

```text
GET  /health
POST /feishu/events
POST /feishu/card-actions
```

内部控制面：

```text
GET  /ready
GET  /version
POST /council/deliberate
POST /tasks/authorize
GET  /threads/{thread_id}
GET  /runs/{run_id}
GET  /approvals/{interrupt_id}
POST /approvals/{interrupt_id}/decision
```

内部控制面必须带：

```text
X-Internal-Admin-Key: <INTERNAL_ADMIN_API_KEY>
```

生产环境不应公开 `/docs`、`/redoc`、`/openapi.json`。

## 飞书后台配置

在飞书开放平台配置：

```text
事件订阅 URL: https://<DOMAIN>/feishu/events
卡片回调 URL: https://<DOMAIN>/feishu/card-actions
```

然后：

1. 邀请机器人进入测试群。
2. 配置机器人收发消息权限。
3. 配置 Bitable app/table 权限。
4. 在 `.env` 填入 `FEISHU_DEFAULT_CHAT_ID`、`FEISHU_BITABLE_APP_TOKEN`、`FEISHU_BITABLE_TABLE_ID`。

## 模型配置

业务模型不再由服务器 `.env` 统一选择。第一版目标是用户在飞书模型配置卡里填写：

```text
provider: openai / deepseek
usage: chat / embedding
model_name
display_name
api_key
base_url 可选
```

服务器 `.env` 只需要：

```env
MODEL_SECRET_ENCRYPTION_KEY=
MODEL_PROVIDER_ALLOWLIST=openai,deepseek
```

`LLM_PROVIDER / LLM_MODEL / LLM_API_KEY` 只作为本地调试 fallback，不属于多人主路径。

## 本地开发

不使用 Docker 时：

```bash
.venv/bin/python -m uvicorn app.main:app --reload
.venv/bin/python -m pytest -q
.venv/bin/python -m ruff check .
```

如果使用本机 PostgreSQL，需要自行设置：

```env
DATABASE_URL=postgresql+psycopg://...
CHECKPOINT_DB_URL=postgresql+psycopg://...
```

Docker Compose 默认会从 `POSTGRES_*` 派生数据库连接。

## Smoke Check

```bash
python3 -m json.tool 项目文件索引.json
sh -n scripts/docker-entrypoint.sh
.venv/bin/python -m ruff check .
.venv/bin/python -m pytest -q
```

服务器可继续执行：

```bash
docker compose exec api lietou-config-check --strict
docker compose exec api lietou-postgres-smoke
```

## 正式使用顺序

1. 服务启动。
2. 飞书后台保存两个回调 URL。
3. 测试群 @机器人发起任务。
4. 系统发送任务确认卡。
5. 用户 approve/edit/reject。
6. approve 后进入 graph。
7. War Room 卡片展示进度和 ReviewGate。
8. 高风险副作用通过飞书卡片人工确认。
9. 人工确认后同步 Bitable。
