# 服务器部署与使用手册

本文按 Feishu First 第一版主链路写：FastAPI API、worker、PostgreSQL + pgvector、Caddy HTTPS、飞书事件/卡片和 Bitable 同步。Discord 只作为 optional adapter。

当前推荐服务器路径是 Linux + Docker Compose。Windows 本机调试仍可用 PowerShell 脚本。

## 1. 部署前准备

服务器需要准备：

- 一台 Linux 服务器，建议 Ubuntu 22.04/24.04。
- 一个域名，A 记录解析到服务器公网 IP。
- 安全组/防火墙放行 `80` 和 `443`。
- Docker Engine 和 Docker Compose v2。
- `git` 和 `python3`。宿主机 Python 只用于运行部署预检；应用依赖在 Docker 镜像内安装。
- 一个飞书开放平台应用、一个测试群、一个 Bitable app。

确认基础命令：

```bash
docker compose version
git --version
python3 --version
```

如果 `docker compose version` 不通，先安装 Docker Engine / Compose v2，再继续。

## 2. 获取代码

推荐放到 `/opt/headhunt-agent`：

```bash
sudo mkdir -p /opt
sudo chown "$USER":"$USER" /opt
git clone https://github.com/YiyuZh/headhunt-agent.git /opt/headhunt-agent
cd /opt/headhunt-agent
```

如果服务器上已经有仓库：

```bash
cd /opt/headhunt-agent
git fetch --prune origin
git checkout main
git pull --ff-only
```

## 3. 生成 `.env`

在项目根目录执行：

```bash
bash scripts/lietou-oneclick.sh
```

这一步会：

- 如果 `.env` 不存在，从 `.env.example` 创建。
- 自动生成 `INTERNAL_ADMIN_API_KEY`、`POSTGRES_PASSWORD`、`MODEL_SECRET_ENCRYPTION_KEY`。
- 检查 Docker、Compose、必填环境变量和占位值。
- 不启动服务，不输出密钥值。

如果只想预检、不创建 `.env`：

```bash
bash scripts/lietou-oneclick.sh --no-init --doctor-only
```

## 4. 填写 `.env`

编辑 `.env`：

```bash
nano .env
```

至少确认这些值：

```env
DOMAIN=your-domain.com
HTTP_PORT=80
HTTPS_PORT=443

INTERNAL_ADMIN_API_KEY=<auto-generated>
MODEL_SECRET_ENCRYPTION_KEY=<auto-generated>

POSTGRES_DB=lietou
POSTGRES_USER=lietou
POSTGRES_PASSWORD=<auto-generated>
POSTGRES_HOST=postgres
POSTGRES_INTERNAL_PORT=5432
POSTGRES_BIND=127.0.0.1
POSTGRES_PORT=5432

CHANNEL_GATEWAY_PROVIDER=feishu
FEISHU_BASE_URL=https://open.feishu.cn
FEISHU_APP_ID=
FEISHU_APP_SECRET=
FEISHU_VERIFICATION_TOKEN=
FEISHU_ENCRYPT_KEY=
FEISHU_DEFAULT_CHAT_ID=
FEISHU_BITABLE_APP_TOKEN=
FEISHU_BITABLE_REQUISITION_TABLE_ID=
FEISHU_BITABLE_CANDIDATE_TABLE_ID=
FEISHU_BITABLE_TALENT_MAP_TABLE_ID=
FEISHU_BITABLE_REPORT_TABLE_ID=

MODEL_PROVIDER_ALLOWLIST=openai,deepseek
EMBEDDING_PROVIDER=openai
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_API_KEY=
```

填写规则：

- `DOMAIN` 必须是你准备用在飞书后台的公网 HTTPS 域名，不要填 `localhost`。
- `INTERNAL_ADMIN_API_KEY`、`POSTGRES_PASSWORD`、`MODEL_SECRET_ENCRYPTION_KEY` 保留自动生成值即可。
- `POSTGRES_BIND=127.0.0.1` 表示数据库只绑定本机，生产环境不要改成 `0.0.0.0`。
- 用户业务模型 Key 不写进共享 `.env`。第一版主路径是用户在飞书模型配置卡里 BYOK。
- `EMBEDDING_API_KEY` 是可选的本地 embedding fallback；没填时长期记忆向量检索会被标记为未就绪。

## 5. 启动前严格预检

```bash
python3 -m app.runtime.local_doctor --strict
```

必须看到 `status` 为 `ok` 才继续。常见 blocker：

- `runtime_required still placeholder`：本地密钥没有生成，重新执行 `bash scripts/lietou-oneclick.sh`。
- `feishu_required missing`：飞书或 Bitable 必填项没填。
- `Install Docker Engine/Compose`：Docker 或 Compose v2 不可用。

## 6. 一键启动

```bash
bash scripts/lietou-oneclick.sh --start
```

这一步会先再次执行 strict preflight，通过后才运行：

```bash
docker compose up -d --build
```

如果只想复用已有镜像，不重新 build：

```bash
bash scripts/lietou-oneclick.sh --start --no-build
```

## 7. 启动后验收

查看容器：

```bash
docker compose ps
```

期望看到 `lietou-api`、`lietou-worker`、`lietou-postgres`、`lietou-caddy` 都是 running/healthy。

查看日志：

```bash
docker compose logs -f api worker caddy postgres
```

检查 HTTP/HTTPS：

```bash
curl -f http://127.0.0.1/health
curl -f https://<DOMAIN>/health
curl -f -H "X-Internal-Admin-Key: <INTERNAL_ADMIN_API_KEY>" https://<DOMAIN>/ready
```

容器内配置和数据库 smoke：

```bash
docker compose exec api lietou-config-check --strict
docker compose exec api lietou-postgres-smoke
docker compose exec api lietou-memory-retention --dry-run
```

单次 worker 检查：

```bash
docker compose exec worker lietou-outbox-worker --once
```

## 8. 飞书后台配置

在飞书开放平台配置：

```text
事件订阅 URL: https://<DOMAIN>/feishu/events
卡片回调 URL: https://<DOMAIN>/feishu/card-actions
```

注意：

- 两个回调入口使用同一组 `Verification Token` 和 `Encrypt Key`。
- `card.action.trigger` 只配置到 `/feishu/card-actions`。
- 不要同时配置旧版消息卡片请求网址或 `card.action.trigger_v1`。
- 回调保存成功不等于业务端到端成功；还要在测试群发真实任务验证卡片发送和点击。

飞书权限至少包括：

- 机器人接收群消息、发送群消息。
- 交互卡片回调。
- Bitable app/table record 读写。

## 9. 真实业务试跑顺序

1. 邀请机器人进入测试群。
2. 在测试群 @机器人发一个简单职位任务。
3. 后端应快速 ACK，并在 outbox 里排队任务确认卡。
4. 你在飞书卡片上 approve/reject。当前 edit 表单仍待实现。
5. approve 后才会写入 `graph_dispatch` outbox，由 worker 进入 graph。
6. 高风险副作用继续通过飞书确认卡走 HumanApproval。
7. 人工确认后才同步 Bitable。

## 10. 常用运维命令

停止服务：

```bash
docker compose down
```

重启并重新 build：

```bash
bash scripts/lietou-oneclick.sh --start
```

拉取最新代码后重启：

```bash
git pull --ff-only
bash scripts/lietou-oneclick.sh --start
```

查看最近日志：

```bash
docker compose logs --tail=200 api worker caddy postgres
```

备份 PostgreSQL 数据卷前，先停服务或使用数据库级备份。当前 Compose 数据在 Docker volume `headhunt-agent_postgres_data` 或项目名对应的 `*_postgres_data` 中。

## 11. 故障排查

### `POSTGRES_PASSWORD is missing`

说明 `.env` 没创建或没有被 Compose 读到。执行：

```bash
pwd
ls -la .env
bash scripts/lietou-oneclick.sh
```

必须在项目根目录运行启动命令。

### `status=not_ready`

看 `blocking_issues`，不要只看最后一行。缺飞书/Bitable 值时，先补 `.env`，再运行：

```bash
python3 -m app.runtime.local_doctor --strict
```

### HTTPS 证书失败

检查：

```bash
dig +short <DOMAIN>
curl -I http://<DOMAIN>/health
docker compose logs caddy --tail=100
```

确认域名解析到服务器公网 IP，云安全组放行 `80/443`，服务器上没有 nginx/apache 占用端口。

### API 不健康

```bash
docker compose logs api --tail=200
docker compose logs postgres --tail=200
docker compose ps
```

常见原因是数据库没 ready、Alembic migration 失败、`.env` 占位符未替换。

### 飞书回调保存失败

先确认公网访问：

```bash
curl -f https://<DOMAIN>/health
```

再确认 `.env` 里的 `FEISHU_VERIFICATION_TOKEN` 和 `FEISHU_ENCRYPT_KEY` 与飞书后台一致。

## 12. Windows 本机调试入口

如果你在 Windows 本机运行：

```powershell
cd D:\apps\headhunt-agent
powershell -ExecutionPolicy Bypass -File scripts\lietou-oneclick.ps1
powershell -ExecutionPolicy Bypass -File scripts\lietou-oneclick.ps1 -Start
```

Windows 本机也要先填 `.env`，并确保 Docker Desktop 已启动。

## 当前未验证边界

- 本机 Windows 已验证 doctor、PowerShell one-click 预检和测试；Linux shell 脚本需要在真实 Linux 服务器上执行验证。
- 真实飞书后台保存回调、真实卡片发送/更新、真实 Bitable 权限和写入，需要你在服务器和飞书后台联调后才能标记为已通过。
- 真实 OpenAI/DeepSeek/embedding 调用需要用户在飞书 BYOK 卡片里保存模型后验证。
- 真实 PostgreSQL checkpointer 在线 interrupt/resume 需要服务启动后走完整 approve/resume 链路验证。
