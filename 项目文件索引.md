## 根目录关键文件

- `AGENTS.md`：Codex 进入项目时的轻量协作规则入口。
- `.gitignore`：忽略系统文件、本地环境、缓存和本地数据库文件。
- `.dockerignore`：Docker build context 排除规则，避免把本地虚拟环境、缓存、secret、测试目录和无关文档打进镜像。
- `.env.example`：环境变量模板，默认 Docker Compose 内部 PostgreSQL + pgvector + PostgreSQL checkpointer + Feishu First 配置占位；Discord 仅为 optional adapter 配置占位，并默认 OpenAI Responses / OpenAI embedding / outbox worker 接入参数。
- `Dockerfile`：FastAPI API 和 durable outbox worker 共用的 Python 3.11 后端镜像。
- `Caddyfile`：Caddy HTTPS 反向代理配置，使用 `DOMAIN` 反代到 `api:8000`。
- `README.md`：GitHub 公开展示用产品首页，包含产品介绍、核心流程、快速启动、普通用户使用方式、常见问题、GitHub 上传步骤和折叠英文 Quick Start。
- `alembic.ini`：Alembic 配置。
- `docker-compose.yml`：服务器一键启动底座，包含 `api`、`worker`、`postgres` 和 `caddy` 服务；真实服务器 `docker compose up -d --build` 尚未验证。
- `pyproject.toml`：Python 3.11+ 项目依赖、`lietou-config-check` / `lietou-external-smoke` / `lietou-outbox-worker` / `lietou-postgres-smoke` 脚本、pytest、ruff 和 setuptools 包发现配置。
- `scripts/docker-entrypoint.sh`：容器启动入口，等待 PostgreSQL 并在 `api` 服务启动前执行 Alembic 迁移。
- `项目prompt.md`：通用项目协作 Prompt 模板，说明任务记忆和文件索引维护要求。
- `任务记忆文档.md`：项目长期协作记忆，后续重要改动必须更新。
- `项目文件索引.md`：人类可读的项目结构索引。
- `项目文件索引.json`：机器可读的项目结构索引。

- 规划 CanonicalTaskBrief：关键字段必须有 `field_source` 和 `confidence`，无 source 推断只能进入 assumptions；double check approve 后冻结 version，下游 Agent 只能读取冻结版。
- Discord 作为 optional adapter；现有 Discord PING、BYOK 和 command register 可保留但不作为第一版主路径。
- 规划记忆主链路：第一版直接实现 `MemoryGateway + VectorMemoryStore + EmbeddingGateway + pgvector`，RunMemory 自动向量化，长期记忆审批后才 active，并按 `30d / 90d / permanent` retention policy 清理、降权、续期或撤销。
- 当前记忆检索边界：MemoryGateway 已按 thread/tenant/guild/user/project/requisition/candidate scope filter fail closed，并在 SQL 中使用兼容过滤：有边界值时只允许同值或 `NULL` 泛化记忆，缺边界值时排除更细粒度记忆；公司当前事实必须由 SearchGateway source_refs 支撑，长期记忆不得替代实时搜索。
- 规划 Agent 上下文：`AgentHarness.build_context_pack` 只注入当前节点需要的 artifact refs、MemoryRef、SOPRef、source refs 和预算信息，禁止每轮注入完整历史、完整 state、完整 node_history、全量 AgentRuns、全量 artifacts 或全量长期记忆。
- 规划审查门：ReviewGate 在关键产物后输出 `pass | needs_fix | needs_human`，自动修复最多一次。
- 规划审查门细则：ReviewGate 是 artifact-level quality gate，必须通过 conditional edge 路由；`needs_fix` 只回对应 repair_node 一次，第二次失败进入人工确认，不得重跑完整 graph。

## Phase 1 工程代码索引

- `app/main.py`：FastAPI 应用工厂和入口。
- `app/api/health.py`：健康、就绪和版本接口。
- `app/api/council.py`：`/council/deliberate` 占位接口，当前通过 PolicyEngine 规则返回 CouncilDecision。
- `app/api/tasks.py`：`/tasks/authorize` 任务授权入口，人工授权后只排队 graph_dispatch outbox。
- `app/api/inspection.py`：`/threads/{thread_id}` 和 `/runs/{run_id}` 只读审计查询入口。
- `app/api/approvals.py`：`/approvals/{interrupt_id}` 和 `/approvals/{interrupt_id}/decision` 内部人工审批入口，只写 HumanApproval + resume outbox。

- `app/api/feishu.py`：飞书事件和卡片回调入口；事件回调和卡片回调使用不同 verifier，ACK 前通过 service 持久化 payload 与 outbox。
- `app/core/config.py`：Pydantic Settings，默认 PostgreSQL + pgvector 配置；当前 runtime readiness 仍保留部分历史 Discord 检查，后续代码阶段需按 Feishu First 收敛；secret 不打印。
