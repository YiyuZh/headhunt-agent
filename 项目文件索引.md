## 1-Line Summary

This is a Python FastAPI + LangGraph AI headhunter runtime: Feishu is the first-version channel, PostgreSQL/pgvector is the persistence and memory base, and Discord remains an optional adapter.

## Architecture Map

### Entrypoints

- app/main.py creates the FastAPI app and mounts app/api/router.py.
- app/api/feishu.py exposes /feishu/events and /feishu/card-actions for Feishu encrypted callbacks.
- app/runtime/local_doctor.py initializes .env, generates local runtime secrets, checks placeholders, Docker, docker compose, and reports OS-appropriate one-click startup commands.
- scripts/lietou-oneclick.sh is the Linux server one-click preflight/start wrapper around local_doctor and docker compose.
- scripts/lietou-oneclick.ps1 is the Windows one-click preflight/start wrapper around local_doctor and docker compose.
- app/runtime/worker.py claims durable feishu_outbox rows and dispatches graph/card/Bitable work.
- app/runtime/graph_factory.py creates headhunter_war_room_graph with optional PostgreSQL checkpointer.
- app/runtime/memory_retention.py runs one-shot memory retention from CLI.

### Data Flow

- Feishu events and card actions are verified with Encrypt Key signatures, optionally decrypted from encrypt payloads, token-checked, persisted, and ACKed quickly.
- Feishu task intake builds a structured task, sends a confirmation card, and only approved tasks enqueue graph_dispatch.
- AgentHarness builds bounded ContextPack with ArtifactRef/MemoryRef/SOPRef metadata, calls the configured LLM gateway, then writes AgentRun, ArtifactStore, and RunMemory outputs.
- Business artifacts pass through ReviewGate before ActionGate can propose side effects.
- HumanApproval approve/edit queues resume; ActionExecutor validates the approval and queues Bitable writes only after approval.

### LangGraph Flow

- receive_user_input -> create_task_plan_and_policy -> council_deliberation_graph -> dispatch_from_council.
- Business subgraphs create RequisitionCalibrationDraft/TalentMapDraft/CandidateMatchDraft/ReportDraft/FollowupReviewDraft/CaseDataDraft refs.
- Each business artifact routes to review_gate; needs_fix repairs once, schema errors fail closed, needs_human/pass continue to human approval before any side effect.
- interrupt_human_approval -> execute_approved_action -> record_agent_run.

### Storage Runtime Boundaries

- PostgreSQL stores events, outbox, graph threads, approvals, artifact refs/blobs, memory items, model profiles, and Bitable sync audit rows.
- pgvector is the first-version memory vector store; MemoryGateway returns MemoryRef summaries only and enforces scope/PII/status filters.
- ArtifactBlob holds full artifact/context payloads; graph state and cards carry summaries and content_ref references.
- FeishuGateway and BitableGateway are called by outbox handlers/workers, not directly by agents.
- User BYOK secrets are encrypted model profiles; War Room/cards/AgentRuns only expose profile metadata.

### Current Unimplemented Or Unverified

- Real Feishu backend callback save, card rendering/submission/update, and War Room card display remain unverified on this machine; encrypted card callback contract is covered locally.
- Task edit forms, model profile list/use/test/revoke cards, richer progress cards, and non-Bitable business side effects are not fully implemented.

- Real Linux server start, PostgreSQL online migration/checkpointer, real Feishu/Bitable permission, real OpenAI/DeepSeek and embedding calls, Docker Compose runtime, and scheduled memory-retention execution remain unverified.
